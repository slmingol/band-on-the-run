## Building the Database

### Automatic Build

To rebuild the entire song database from scratch:

```bash
cd /Users/smingolelli/dev/projects/band-on-the-run
node scripts/build-comprehensive-song-database.js
```

This script:
1. Loads the existing Billboard #1 hits from `scripts/top-songs.json`
2. Adds songs from all 47 categories
3. Deduplicates using lowercase `title|||artist` keys
4. Outputs final count and statistics
5. Saves to `scripts/top-songs.json`

### Deduplication

The database uses smart deduplication to prevent duplicate songs:

```javascript
function deduplicateSongs(songs) {
  const seen = new Set();
  return songs.filter(song => {
    const key = `${song.title.toLowerCase()}|||${song.artist.toLowerCase()}`;
    if (seen.has(key)) return false;
    seen.add(key);
    return true;
  });
}
```

This ensures that songs appearing in multiple categories are only included once.

## Database Format

Each song entry follows this format:

```json
{
  "title": "Song Title",
  "artist": "Artist Name"
}
```

The database is stored as a JSON array in `scripts/top-songs.json` and `server/data/top-songs.json`.

## Adding Songs

The database is designed to be comprehensive and generally doesn't need manual additions. However, to add more songs:

1. Edit `scripts/build-comprehensive-song-database.js`
2. Add a new category section with songs
3. Push songs to the `allSongs` array
4. Run the build script
5. Copy to `server/data/top-songs.json`

Example:

```javascript
// New Category
console.log('🎵 Adding [category name]...');
const newCategory = [
  { "title": "Song 1", "artist": "Artist 1" },
  { "title": "Song 2", "artist": "Artist 2" },
];
allSongs.push(...newCategory);
```

## Maintenance

### Updating Categories

To modify or replace a category:

1. Open `scripts/build-comprehensive-song-database.js`
2. Find the category section (search for the emoji or console.log)
3. Update the song array
4. Rebuild the database
5. Copy to server/data if needed
6. Commit changes
